# 畳み込み符号 誤り訂正能力評価プロジェクト

このディレクトリは、情報通信論の課題である「1ビット入力2ビット出力の畳み込み符号において遅延素子の個数に対する誤り訂正能力の評価」を行うためのコード一式です。3種類の畳み込み符号器（遅延素子数 m=1, 2, 3）と、それに対応する硬判定ビタビ復号器をC言語でフルスクラッチ実装し、評価を行っています。

## ディレクトリ構成

- `c/`: 畳み込み符号器、トレリス構築、ビタビ復号器などのC言語ソースコード群と `Makefile` が格納されています。
- `python/`: 実行結果のCSVデータを読み込み、評価グラフを生成するPythonスクリプト（`plot_results.py`）が含まれています。
- `output/`: プログラム実行時に生成される実験結果データ（`results.csv`）やグラフ画像（`error_correction_results.png`）の出力先です。

## 実行方法

このプロジェクトの評価実験は、以下のステップで実行します。

### 1. プログラムのビルドと評価実行（メイン処理）
`c/` ディレクトリに移動し、`make` コマンドでビルド後に評価プログラムを実行します。
```bash
cd c/
make clean
make
./error_eval --verbose
```
※ コマンドライン引数 `--seed <数値>` で乱数シードを固定したり、`--max-errors <数値>` で評価する最大誤り数を変更することも可能です。

### 2. グラフの生成
評価実行後、出力されたCSVデータをもとにPythonスクリプトでグラフを生成します。
```bash
cd ../python
python3 plot_results.py
```

## 実行すると何が起きるか（確認できる結果）

`error_eval` および `plot_results.py` を実行すると、以下の処理が自動で行われます。

1. **自動検証 (ラウンドトリップテスト)**: 評価に入る前に、生成した100ビットの乱数情報に対して「符号化 → 誤り付加なし → ビタビ復号」の完全なパイプラインを実行し、情報が1ビットの狂いもなく復元できるか（アルゴリズムの正当性）を全符号器で自動テストします。
2. **ハイブリッド試行による厳密な評価**: 符号語に付加する誤り数を1から徐々に増やし、訂正成功率を計算します。
   - 誤り数が少ない場合（e=1, 2）は、**全パターン列挙**を行い、厳密な成功数をカウントします。
   - 誤り数が多い場合（e≧3）は、Fisher-Yatesシャッフルを用いたランダム抽出で指定回数（デフォルト100回）テストを行い、分散を抑えた成功率を導き出します。
3. **無駄な処理の自動打ち切り**: 誤り限界を超え、訂正成功率 0% が3回連続で続いた場合は、その符号器の評価を打ち切り、高速に処理を完了させます。

### 出力される成果物

ターミナル上に実行の進捗と結果テーブルが表示され、`output/` ディレクトリ内に以下のファイルが生成されます。

- **`results.csv`**: 各符号器の評価結果が記録されたデータ。
  - **encoder**: 符号器の名前（G1, G2, G3）
  - **n_delay**: 遅延素子の数 (m)
  - **num_errors**: 符号語に付加した誤りの数
  - **num_trials / num_success / num_fail**: 試行回数、訂正成功回数、失敗回数
  - **success_rate**: 訂正成功率
- **`error_correction_results.png`**: CSVデータをもとに描画された、遅延素子数ごとの「誤り数 vs 訂正成功率」のグラフ画像。レポートの考察にそのまま使用できます。
