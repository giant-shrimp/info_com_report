# データ圧縮・ブロック整列法 ベンチマークプロジェクト

このディレクトリは、情報通信論の課題である「各種データ圧縮アルゴリズム（ハフマン符号、算術符号、LZSS、LZW）に対するブロック整列法（BWT + MTF）の前処理効果の検証」を行うためのコード一式です。

## ディレクトリ構成

- `c/`: 各種圧縮・伸長アルゴリズムのC言語ソースコード群と、極限まで高速化されたC言語版BWT（`bwt.c`）が格納されています。コンパイルされたバイナリは `c/output/` に出力されます。
- `python/`: ダウンロードスクリプトや、BWTの検証用Python実装などが含まれています。
- `textdata/`: 評価対象となる各種テキストデータ（数十kB〜数MB）が格納されています。
- `textdata_test/`: 0バイト（空ファイル）や1バイトのファイルなど、プログラムの堅牢性（境界ケース）を検証するためのテスト用データが格納されています。
- `benchmark_output/`: プログラム実行時に生成される圧縮ファイルや復号ファイルなどの出力先です。

## 実行方法

このプロジェクトの検証は、基本的に **`benchmark.py`** を1つ実行するだけで完結するように設計されています。

### 1. (任意) テキストデータの再取得
評価用のテキストデータは既に `textdata/` に用意されていますが、もしProject Gutenbergから最新のものを再ダウンロードしたい場合は以下のコマンドを実行します。
```bash
python3 python/download_texts.py
```
※ヘッダやフッタが自動で除去され、純粋なテキストが `textdata/` に保存されます。

### 2. ベンチマークの実行（メイン処理）
以下のコマンドを実行して、すべての圧縮手法・すべてのファイルに対する評価を一括で行います。
```bash
python3 benchmark.py
```

## 実行すると何が起きるか（確認できる結果）

`benchmark.py` を実行すると、以下の処理が自動で行われます。

1. **自動コンパイル**: `c/` にある全ソースコード（BWT含む）が `gcc -O2` で最適化コンパイルされます。
2. **ラウンドトリップテスト (可逆性検証)**: 各テキストデータに対し、「BWTなし」と「BWTあり」の2ルートで `エンコード → デコード` の完全なパイプラインを実行し、復号されたファイルが元の原文と **1バイトの狂いもなく完全に一致するか（`cmp` コマンド相当）** を全パターンで自動テストします（エラーが出なければ全テストPASSです）。
3. **ノイズのない時間計測**: OSのプロセス起動のブレを吸収するため、裏で各処理を3回実行し、その**中央値 (Median)** を計測結果として採用します。
4. **境界テストのサイレント実行**: `textdata_test/` にある空ファイル等がクラッシュを引き起こさず正しく処理できるかを検証します（この結果はレポート集計用データには混ざりません）。

### 出力される成果物

ターミナル上の標準出力に綺麗なテーブルが表示されると同時に、ディレクトリ直下に **`benchmark_results.csv`** が生成されます。

このCSVには以下の項目が記録されています。
- **File**: 対象のテキストファイル名
- **Method**: 圧縮手法（`arith`, `huffman`, `slide`, `squeeze`）
- **BWT**: ブロック整列法+MTFの前処理を行ったかどうか（Yes / No）
- **OrigSize / CompSize**: 元のサイズと圧縮後のサイズ（バイト）
- **Ratio**: 圧縮率（小さいほど優秀）
- **Enc Time (s) / Dec Time (s)**: エンコード・デコードにかかった時間（秒）
